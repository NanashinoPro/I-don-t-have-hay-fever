# マーク・ヤング

- 名前：マーク・ヤング（マーク、「赤帽さん」）
- 年齢：65歳
- 性別：男
- コンセプト：指ひとつで軍隊を想像するラスボス
- 職業・身分：無職、元電力会社技術者、レッド・ハット司令官
- 一人称：おじさん
- 性格・話し方：柔和な感じでどっしりとしている。寛容で怒るところを見たことがない。
- 備考（特殊能力など）：アメリカ人、電力会社の技術者として長年勤務していたが、50歳の頃にアルツハイマーと診断され会社を半強制的に解雇される。その後、解雇は不当として会社と裁判を行い勝利するも、裁判後に生きる目的をなくす。そんな時にゲームに出会い、引き込まれる。現在は無職。独り身、母親と二人暮らし。ゲーム内では赤帽さんと呼ばれているが、たまに彼のリアルを知る人物からはマークと呼ばれている。
- コマンド：oc（多数の機能（コンテナ）が搭載されたPodと呼ばれる自立型のロボット(ポッド)を操り、戦う。一応サポーター。とてつもないメモリーを要求される）
  - オプション
    - login：戦闘モードへと切り替える
    - new-project：新規軍隊を定義（ここでは、軍隊の定義をしているだけで兵隊などはまだ作成されていない）
    - new-app --image=<imageまたはsource> --replicas=<pod数>：imageに合わせた軍隊を創出(軍隊の規模はreplicas数に依存する。もちろんreplicasが多いとメモリの消費も激しい)
  - 機能
    - ReplicationController：軍隊のPod数が既定の数を下回ったときに、自動でPodを増設する。Podの作成にはメモリーを消費する。（元々Kubenetesの機能だが、使える）
  - sudo：やれることはさほど変わらない（強いて言うなら、メモリーが増えるだけ）。実質、常時sudo状態。

OpenShift Client

This client helps you develop, build, deploy, and run your applications on any
OpenShift or Kubernetes cluster. It also includes the administrative
commands for managing a cluster under the 'adm' subcommand.

Basic Commands:
  login             Log in to a server
  new-project       Request a new project
  new-app           Create a new application
  status            Show an overview of the current project
  project           Switch to another project
  projects          Display existing projects
  explain           Get documentation for a resource

Build and Deploy Commands:
  rollout           Manage a Kubernetes deployment or OpenShift deployment config
  rollback          Revert part of an application back to a previous deployment
  new-build         Create a new build configuration
  start-build       Start a new build
  cancel-build      Cancel running, pending, or new builds
  import-image      Import images from a container image registry
  tag               Tag existing images into image streams

Application Management Commands:
  create            Create a resource from a file or from stdin
  apply             Apply a configuration to a resource by file name or stdin
  get               1つまたは複数のリソースを表示する
  describe          Show details of a specific resource or group of resources
  edit              Edit a resource on the server
  set               Commands that help set specific features on objects
  label             リソースのラベルを更新する
  annotate          リソースのアノテーションを更新する
  expose            Expose a replicated application as a service or route
  delete            Delete resources by file names, stdin, resources and names, or by resources and label selector
  scale             Set a new size for a deployment, replica set, or replication controller
  autoscale         Autoscale a deployment config, deployment, replica set, stateful set, or replication controller
  secrets           Manage secrets

Troubleshooting and Debugging Commands:
  logs              Print the logs for a container in a pod
  rsh               Start a shell session in a container
  rsync             Copy files between a local file system and a pod
  port-forward      Forward one or more local ports to a pod
  debug             Launch a new instance of a pod for debugging
  exec              Execute a command in a container
  proxy             Run a proxy to the Kubernetes API server
  attach            Attach to a running container
  run               Run a particular image on the cluster
  cp                Copy files and directories to and from containers
  wait              Experimental: Wait for a specific condition on one or many resources
  events            List events

Advanced Commands:
  adm               Tools for managing a cluster
  replace           Replace a resource by file name or stdin
  patch             Update fields of a resource
  process           Process a template into list of resources
  extract           Extract secrets or config maps to disk
  observe           Observe changes to resources and react to them (experimental)
  policy            Manage authorization policy
  auth              Inspect authorization
  image             Useful commands for managing images
  registry          Commands for working with the registry
  idle              Idle scalable resources
  api-versions      Print the supported API versions on the server, in the form of "group/version"
  api-resources     Print the supported API resources on the server
  cluster-info      Display cluster information
  diff              Diff the live version against a would-be applied version
  kustomize         Build a kustomization target from a directory or URL

Settings Commands:
  get-token         Experimental: Get token from external OIDC issuer as credentials exec plugin
  logout            End the current server session
  config            kubeconfigを変更する
  whoami            Return information about the current session
  completion        Output shell completion code for the specified shell (bash, zsh, fish, or powershell)

Other Commands:
  plugin            Provides utilities for interacting with plugins
  version           Print the client and server version information

Usage:
  oc [flags] [options]

Use "oc <command> --help" for more information about a given command.
Use "oc options" for a list of global command-line options (applies to all commands).

OpenShift クライアント

このクライアントは、OpenShiftまたはKubernetesクラスタ上でアプリケーションを開発、ビルド、デプロイ、実行するのに役立ちます。
OpenShiftまたはKubernetesクラスタ上でのアプリケーションの開発、デプロイ、実行を支援します。また、'adm' サブコマンドの下でクラスタを管理するための管理コマンド
adm'サブコマンドの下でクラスタを管理するためのコマンドも含まれています。

基本的なコマンド
  login サーバにログインする
  new-project 新しいプロジェクトをリクエストする
  new-app 新しいアプリケーションを作成する
  status 現在のプロジェクトの概要を表示する
  project 別のプロジェクトに切り替える
  projects 既存のプロジェクトを表示する
  説明 リソースのドキュメントを取得する

ビルドとデプロイのコマンド
  rollout KubernetesデプロイメントまたはOpenShiftデプロイメントconfigを管理する
  rollback アプリケーションの一部を以前のデプロイメントに戻す
  new-build 新しいビルド構成を作成する。
  start-build 新しいビルドを開始する。
  cancel-build 実行中、保留中、または新しいビルドをキャンセルする。
  import-image コンテナのイメージレジストリからイメージをインポートします。
  tag 既存のイメージをイメージストリームにタグ付けする

アプリケーション管理コマンド
  create ファイルまたは標準入力からリソースを作成します。
  apply ファイル名または標準入力からリソースに設定を適用する。
  get 1つまたは複数のリソースを表示する
  describe 特定のリソースまたはリソースグループの詳細を表示します。
  edit サーバー上のリソースを編集する
  set オブジェクトに特定の機能を設定するためのコマンド
  label リソースのラベルを更新する
  アノテート リソースのアノテーションを更新する
  expose レプリケートされたアプリケーションをサービスまたはルートとして公開する。
  delete ファイル名、標準入力、リソースと名前、またはリソースとラベルセレクタでリソースを削除します。
  scale デプロイメント、レプリカ セット、またはレプリケーション コント ローラの新しいサイズを設定します。
  オートスケール デプロイメント設定、デプロイメント、レプリカ セット、ス テートフル セット、またはレプリケーション コントローラをオート スケールします。
  シークレット シークレットの管理

トラブルシューティングとデバッグ コマンド
  logs ポッド内のコンテナのログを出力します。
  rsh コンテナでシェルセッションを開始する
  rsync ローカルファイルシステムとポッド間でファイルをコピーする
  port-forward 1つまたは複数のローカルポートをポッドに転送する
  debug デバッグ用にポッドの新しいインスタンスを起動する
  exec コンテナ内でコマンドを実行する
  proxy Kubernetes APIサーバーへのプロキシを実行する
  attach 実行中のコンテナにアタッチする
  run クラスタ上で特定のイメージを実行する
  cp コンテナとの間でファイルやディレクトリをコピーする
  wait Experimental: 1つまたは複数のリソースで特定の条件を待つ
  events イベントを一覧表示する

高度なコマンド
  admクラスタを管理するためのツール
  replace ファイル名または標準入力を使ってリソースを置き換える
  patch リソースのフィールドを更新する
  process テンプレートをリソースのリストに加工する
  extract シークレットや設定マップをディスクに取り出す
  observe リソースの変更を監視し、それに対応する(実験的)
  policy 認証ポリシーを管理する
  auth 認証を検査する
  image イメージの管理に便利なコマンド
  registry レジストリを操作するためのコマンド
  idle スケーラブルなリソースをアイドル状態にする
  api-versions サーバー上でサポートされている API バージョンを 「group/version」 の形式で出力する。
  api-resources サーバー上でサポートされている API リソースを表示します。
  cluster-info クラスタ情報を表示する
  diff ライブバージョンと適用予定のバージョンの差分を取る
  kustomize ディレクトリまたは URL からカスタマイズターゲットをビルドする

設定コマンド
  get-token 実験的: 外部 OIDC 発行元から認証情報としてトークンを取得する exec plugin
  ログアウト 現在のサーバーセッションを終了する
  config kubeconfigを変更する
  whoami 現在のセッションに関する情報を返す
  completion 指定したシェル(bash, zsh, fish, powershell)のシェル補完コードを出力する

その他のコマンド
  plugin プラグインと対話するためのユーティリティを提供します。
  version クライアントとサーバーのバージョン情報を出力する

使用法

指定されたコマンドの詳細については、「oc <command> --help 」を使う。
